需求背景

当前 CRM 系统中线索数据缺少灵活的标签分类体系，各业务线（业务线A、业务线B、业务线C、业务线D、业务线E）无法对线索进行精细化运营分类。 业务侧反馈需要在后台系统中建立统一的标签管理机制，支持多级标签分类、标签颜色区分，并在线索池列表中便捷地为线索打标签， 以提升线索筛选效率和精准营销能力。

功能全景

本次迭代在 CRM 模块下新增"线索标签管理"菜单，包含以下核心功能：

1. 标签组管理 — 管理标签的顶层分组（如：标签组A、标签组B、标签组C），支持增删改查和排序。
2. 标签管理 — 在标签组下管理一级标签和二级标签，支持颜色配置、启用/停用、关联产线、触发事件配置。
3. 线索池打标签 — 在各业务线线索池列表中，支持单条/批量为线索添加或移除标签。
5. 标签检索线索 — 跨产线全局检索，通过组合标签条件快速定位目标线索集合，含统计视图。
6. 标签操作日志 — 全局记录所有标签操作历史（操作人、工号、操作类型、时间等），支持筛选和导出。
7. 标签统计 — 记录标签使用次数、使用产线分布，辅助标签治理。

业务规则与约束
规则项说明
标签组名称唯一性同一系统内标签组名称 不可重复，编码也不可重复
标签名称唯一性同一标签组内，一级标签名称 不可重复；同一级标签下，二级标签名称不可重复
标签组停用联动停用标签组时，其下所有标签同步停用，已打标签的线索数据保留，但标签不再可选
一级标签停用联动停用一级标签时，其下所有二级标签同步停用
删除保护已被线索使用的标签 不可删除，只可停用；未使用的标签可物理删除

V1.0.0 > 需求概述

业务标签管理 — 需求概述

1.1 需求背景

当前 CRM 系统中线索数据缺乏灵活的标签分类能力。各业务线（业务线A、业务线B、业务线C、业务线D、业务线E）运营人员无法根据标签组A、标签组B、标签组C等维度对线索进行精细化分类，导致线索筛选效率低、精准营销能力不足。

业务侧提出在后台管理系统（管理后台）的 CRM 模块下，新增"线索标签管理"功能，建立统一的标签体系和打标能力。

1.2 需求目标

1. 建立统一的标签管理体系，支持标签组 → 一级标签 → 二级标签的三级结构。

2. 支持标签颜色配置，便于视觉区分。

3. 支持标签与产线的关联配置，实现产线级别的标签隔离。

4. 在各业务线线索池列表中新增标签列，支持单条和批量打标签操作。

5. 支持按标签筛选线索，提升线索检索效率。

6. 统计标签使用次数和使用产线分布，辅助标签治理和优化。

7. 提供跨产线的全局标签检索能力，通过组合标签条件快速定位目标线索集合。

1.3 功能范围

新增菜单：CRM → 线索标签管理（下设"标签组管理"、"标签管理"和"标签检索线索"三个子菜单）

改动页面：5个线索池列表页（业务线A/业务线B/业务线C/业务线D/业务线E）均新增标签列和操作入口

新增数据表：tag_group（标签组）、tag（标签）、tag_product_line（标签-产线关联）、lead_tag_relation（线索-标签关联）

本需求不涉及标签的自动打标逻辑（如根据行为自动打标签），仅支持手动打标。自动打标能力作为后续迭代规划。
1.4 用户角色

CRM管理员 / 超级管理员：负责标签体系的搭建和维护（标签组/标签的增删改、启停操作）。

产线经理：在线索池中使用标签对线索进行分类。

客服/销售人员：在线索池中使用标签，但不具备标签体系管理权限和批量打标权限。

3. 选中一级标签时自动包含其下所有二级标签

筛选逻辑：

同一标签组内多个标签为 OR（或）关系 — 线索命中任一标签即符合

标签检索线索

6.1 功能定位

标签检索线索是一个跨产线的全局线索检索页面，允许用户通过组合标签条件快速定位目标线索集合，突破单一产线线索池的限制。

菜单路径：CRM → 线索标签管理 → 标签检索线索

与各线索池列表中的"标签筛选"不同，本页面的核心价值在于：
1. 跨产线检索 — 不再局限于某个产线，可一次检索所有产线中符合标签条件的线索
2. 多维组合 — 支持多个标签组、多级标签的复杂组合条件
3. 统计视图 — 提供匹配结果的产线分布等聚合统计
6.2 检索条件区域

页面顶部为检索条件区域，包含以下筛选项：

所属产线：下拉选择，限定检索范围。选择"全部产线"时跨产线检索，选择具体产线时仅检索该产线线索。
标签组筛选：下拉选择标签组，缩小标签选择范围。
选择标签：标签输入框，支持搜索和选择多个标签。已选标签以带颜色的 tag 形式展示在输入框中，可点击 × 移除。

6.3 匹配逻辑切换

提供两种匹配逻辑切换：

AND（同时满足）：线索必须同时拥有所有已选标签才命中。适用于精确筛选场景，如"一级标签A 且 抖音广告"。
OR（满足任一）：线索拥有任一已选标签即命中。适用于扩大范围场景，如"一级标签A 或 一级标签B"。

默认匹配逻辑为 AND。

标签组内的 OR/AND 逻辑仅作用于全局检索页。各产线线索池列表中的"标签筛选"仍保持原有逻辑（组内OR、组间AND）。
6.4 检索结果统计卡片

检索条件区域下方展示 2 个统计卡片，实时反映匹配结果的聚合数据：

匹配线索总数：符合当前标签条件的线索总条数
覆盖产线数：匹配线索涉及了多少个产线

6.5 检索结果列表

列表展示字段：手机号、所属产线、标签、线索来源、最后跟进时间、操作。

操作列包含：详情、打标签。点击"打标签"弹出与线索池中一致的打标签弹窗。

结果列表支持：

1. 按产线二次筛选（列表内筛选栏）
2. 勾选线索后批量打标签（复用线索池打标签弹窗）
3. 导出结果为 Excel 文件

6.6 产线分布统计视图

在检索结果列表下方，以卡片形式展示匹配线索在各产线的分布情况。每张卡片显示：

产线名称、匹配线索数、占总数百分比、进度条可视化

仅显示有匹配线索的产线卡片，无匹配数据的产线不展示。

接口方法路径说明

标签操作日志

7.1 功能定位

标签操作日志是一个全局只读查询页面，记录系统中所有标签相关操作的完整历史，用于运营追溯、审计和数据分析。

菜单路径：CRM → 线索标签管理 → 标签操作日志

7.2 记录范围

以下操作均会产生日志记录：

1. 线索打标 — 单条/批量为线索添加标签（来源：线索池手动、批量打标）
2. 线索移除标签 — 单条/批量移除线索标签（来源：线索池手动、批量移除）
3. 标签配置变更 — 新建/编辑/启停标签组和标签（记录配置操作）
4. 触发事件执行 — 标签触发事件（如标记线索无效）的系统自动操作

7.3 日志字段定义
字段类型说明
操作时间datetime精确到秒，如 2026-07-01 14:32:08
操作人string操作人姓名
工号string格式示例：(80012345)，8位数字
操作类型enum添加标签 / 移除标签 / 标签配置变更 / 触发事件
所属产线string线索所属产线（业务线A/业务线B/业务线C/业务线D/业务线E）
线索手机号string脱敏显示，如 138****5678
标签名称string被操作的标签名，多个标签以逗号分隔
操作来源enum线索池手动 / 批量打标 / 标签管理 / 系统触发
触发事件产生的日志需以特殊样式标记（如红色背景行），便于快速识别"线索被标记无效"的操作链路，追溯到具体操作人和标签。
7.4 筛选条件

操作类型：下拉选择，支持多选（添加标签/移除标签/标签配置变更/触发事件）
所属产线：下拉选择，支持多选
操作人工号/姓名：模糊搜索
线索手机号：精确/模糊搜索
标签名称：模糊搜索
操作时间范围：日期区间选择器，默认最近30天

7.5 数据模型
表核心字段说明
tag_operation_logid, operator_name, operator_no, operation_type, product_line, lead_phone, tag_names, source, triggered_event, created_at标签操作日志表（新增）
7.6 接口设计
接口方法路径说明
查询操作日志GET/api/crm/tag-log/list分页查询，支持多条件筛选
导出操作日志GET/api/crm/tag-log/export返回 Excel 文件流，上限10000条
7.7 日志写入时机

日志在以下接口调用成功后自动写入：

POST /api/crm/lead/add-tags — 每个标签×每条线索生成一条日志
POST /api/crm/lead/remove-tags — 同上
POST/PUT /api/crm/tag-group/* — 标签组增删改
POST/PUT /api/crm/tag/* — 标签增删改
触发事件执行时 — 额外写入一条"触发事件"类型日志

7.8 权限要求

标签操作日志页面需拥有 crm:tag:log:menu 权限点。建议默认赋予超级管理员、CRM管理员角色。产线经理可视需要开放。

日志数据建议保留至少 180天，超期数据可归档至冷存储，不影响主表查询性能。

7.5 权限控权点

本次需求共需标记 11 个权限控权点（菜单 + 按钮），开发只需在对应 UI 元素上做权限标识，上线后在系统角色权限模块中按需配置即可：

1. crm:tag:menu — 线索标签管理菜单可见性
2. crm:tag:group:create — 新建标签组按钮
3. crm:tag:group:edit — 标签组编辑/启停操作
4. crm:tag:tag:create — 新建标签按钮
5. crm:tag:tag:edit — 标签编辑/启停操作
6. crm:lead:tag:batch — 批量打标签/批量移除标签按钮
7. crm:tag:search:menu — 标签检索线索菜单可见性
8. crm:tag:search:export — 标签检索结果导出按钮
9. crm:tag:trigger:edit — 编辑二级标签弹窗"触发事件"字段
10. crm:tag:log:menu — 标签操作日志菜单可见性
11. crm:lead:firstcall:restore — 标签"恢复为有效"按钮

未在清单中的操作（如查看列表、单条打标签、标签筛选）默认可见，无需权限控制。
7.6 数据迁移

本需求为全新功能，不涉及历史数据迁移。上线后由 CRM 管理员初始化标签体系即可。

7.7 后续迭代方向

1. 自动打标：根据线索行为（如访问页面、咨询关键词）自动关联标签

2. 标签画像：基于标签分布生成客户群体画像分析

3. 标签导入导出：支持 Excel 批量导入标签配置、导出线索标签数据

4. 标签有效期：支持设置标签自动过期时间
